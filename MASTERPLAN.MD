# MASTERPLAN

## 1. Proje Vizyonu ve Amacı
Bu projenin amacı, Mackolik üzerinden geçmiş maç verilerini ve bahis oranlarını toplayarak **"Benzer oranlı maçların geçmişte nasıl sonuçlandığını"** anlaşılır şekilde sunan bir analiz platformu oluşturmaktır.

- **Temel Metot:** Oran-temelli benzerlik (exact / toleranslı eşleşme).
- **Hedef:** Oran dağılımlarını ve olası sonuç eğilimlerini göstermek.
- **Kapsam:** Majör ligler + aktif sezonlar + LiveData fikstürleri.
- **ML Hazırlığı:** Maç istatistikleri (şut, korner, possession) ve H2H verileri toplanarak gelecekte ML modeli eğitimi için veri tabanı oluşturulur.
- **ML Pipeline:** 5 bahis marketi (MS, KG, AU 2.5, TG, İY) için LightGBM modelleri ile value bet sinyalleri.

---

## 2. Sistem Bileşenleri
- **Veritabanı:** Supabase (PostgreSQL).
- **Scraper CLI (Python):** `main.py` üzerinden tüm operasyonlar.
- **Scraping:** Playwright (browser) + Scrapling (Adaptor parse + Fetcher HTTP). Wrapper: `scraping_client.py`.
- **Queue & Worker:** `match_queue` tablosu + `batch_processor.py`.
- **Fikstür Takip:** `livedata_update_fixtures.py` + `monitoring_worker.py`.
- **H2H & Stats:** `h2h_scraper.py` + `scraper_engine.py:parse_match_stats()`.
- **Backfill:** `backfill.py` — geçmiş maçlar için stats + H2H doldurma.
- **Web UI:** Next.js (`web-ui/`) + Supabase istemcisi.

## 3. Mimari Akışlar (Workflow)

### 3.1 Geçmiş Veri Toplama
1) `fill-queue` aktif sezonlardan maç linklerini kuyruğa ekler.
2) `run-worker` Playwright ile maçı işler ve `matches` tablosuna upsert yapar.
3) Aynı anda `match_stats` ve `match_h2h` tablolarına istatistik/H2H verileri yazılır.
4) `match_queue.status` `SUCCESS / BAD_DATA / ERROR` olarak güncellenir.

### 3.2 Fikstür Senkronizasyonu + Monitoring
1) `update-fixtures` LiveData üzerinden gelecek maçları kuyruğa `MONITORING` olarak yazar.
2) `run-monitoring-worker` ±24 saat penceresinde maçları kontrol eder.
3) Skor geldiyse `SUCCESS`, aksi halde `MONITORING` kalır.

### 3.3 Stats + H2H Veri Toplama
1) Her `process_full_match()` çağrısında, skor varsa `parse_match_stats()` çalışır → `match_stats` tablosuna upsert.
2) Her maç için `scrape_h2h()` çalışır → `match_h2h` tablosuna upsert.
3) Stats/H2H hataları ana maç verisini etkilemez (ayrı try-except).
4) `backfill` komutu ile geçmiş maçlar için toplu doldurma yapılabilir.

### 3.4 ML Tahmin Pipeline
1) `train-models` tüm veriyi Supabase'den çeker (matches + match_stats + match_h2h).
2) Feature engineering: ~93 feature (oran implied prob, H2H, rolling takım istatistikleri, bağlamsal).
3) Temporal split (train %80, val %10, test %10) ile LightGBM modelleri eğitilir.
4) 5 market: MS (1/X/2), KG (Var/Yok), AU 2.5 (Alt/Üst), TG (0-1/2-3/4+), İY (1/X/2).
5) İY/MS = İY modeli × MS modeli olasılık matrisi (9 hücre, bağımsızlık varsayımı).
6) Modeller `ml_models/` dizinine kaydedilir.
7) `predict` gelecek maçlar için tahmin üretir, value bet sinyalleri (edge > %5) tespit eder.
8) Tahminler `ml_predictions` tablosuna yazılır.

### 3.5 Analiz & Web UI
- `matches` tablosu tek kaynak gerçektir (fikstür + geçmiş).
- Oran benzerliği (toleranslı) `/match/[id]` ve `/analysis/[id]` ile sunulur.
- Tam eşleşme akışı `/perfect-match` ekranında gösterilir.

## 4. Veritabanı Şeması (Güncel)
**Tek kaynak tablo:** `matches`
**Not:** `weekly_fixtures` deprecated (legacy).

### A. Referans Tabloları
**1) `leagues`**
```sql
CREATE TABLE leagues (
  id SERIAL PRIMARY KEY,
  name VARCHAR(100) NOT NULL,
  country VARCHAR(50),
  created_at TIMESTAMP DEFAULT NOW()
);
```

**2) `seasons`**
```sql
CREATE TABLE seasons (
  id SERIAL PRIMARY KEY,
  league_id INT REFERENCES leagues(id),
  season_year VARCHAR(20) NOT NULL,
  mackolik_id INT NOT NULL UNIQUE,
  total_weeks INT DEFAULT 38,
  is_active BOOLEAN DEFAULT TRUE
);
```

### B. Operasyon Tabloları
**3) `match_queue`**
```sql
CREATE TABLE match_queue (
  match_code VARCHAR(50) PRIMARY KEY,
  match_url TEXT,
  season_id INT,
  season_name VARCHAR(100),
  week INT,
  status VARCHAR(20) DEFAULT 'PENDING', -- PENDING, MONITORING, SUCCESS, ERROR, BAD_DATA
  error_log TEXT,
  last_try_at TIMESTAMP
);
```

**4) `matches`**
```sql
CREATE TABLE matches (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  match_code VARCHAR(50) UNIQUE,
  home_team VARCHAR(100),
  away_team VARCHAR(100),
  league VARCHAR(100),
  season VARCHAR(50),
  match_date TIMESTAMP,

  -- SKOR / DURUM
  score_ft VARCHAR(20),
  score_ht VARCHAR(20),
  status VARCHAR(20), -- MS / FIXTURE / Ert vb.

  -- MS 1/X/2
  ms_1 FLOAT, ms_x FLOAT, ms_2 FLOAT,

  -- Çifte Şans
  cs_1x FLOAT, cs_12 FLOAT, cs_x2 FLOAT,

  -- İY/MS
  iyms_11 FLOAT, iyms_1x FLOAT, iyms_12 FLOAT,
  iyms_x1 FLOAT, iyms_xx FLOAT, iyms_x2 FLOAT,
  iyms_21 FLOAT, iyms_2x FLOAT, iyms_22 FLOAT,

  -- ALT/ÜST & KG
  au_15_alt FLOAT, au_15_ust FLOAT,
  au_25_alt FLOAT, au_25_ust FLOAT,
  kg_var FLOAT, kg_yok FLOAT,

  -- GOL ARALIĞI
  tg_0_1 FLOAT, tg_2_3 FLOAT, tg_4_5 FLOAT, tg_6_plus FLOAT
);
```

### C. İstatistik Tabloları
**5) `match_stats`**
```sql
CREATE TABLE match_stats (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  match_code VARCHAR(50) UNIQUE NOT NULL REFERENCES matches(match_code),
  shots_home INT, shots_away INT,
  shots_on_home INT, shots_on_away INT,
  corners_home INT, corners_away INT,
  possession_home FLOAT, possession_away FLOAT,
  fouls_home INT, fouls_away INT,
  offsides_home INT, offsides_away INT,
  scraped_at TIMESTAMP DEFAULT NOW()
);
```

**6) `match_h2h`**
```sql
CREATE TABLE match_h2h (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  match_code VARCHAR(50) UNIQUE NOT NULL REFERENCES matches(match_code),
  form_home VARCHAR(10), form_away VARCHAR(10),
  standing_home INT, standing_away INT,
  points_home INT, points_away INT,
  tg_dist_home JSONB, tg_dist_away JSONB,
  kg_pct_home FLOAT, kg_pct_away FLOAT,
  au25_over_pct_home FLOAT, au25_over_pct_away FLOAT,
  scorers_home JSONB, scorers_away JSONB,
  referee_stats JSONB,
  h2h_total JSONB,
  scraped_at TIMESTAMP DEFAULT NOW()
);
```

### D. ML Tabloları
**7) `ml_predictions`**
```sql
CREATE TABLE ml_predictions (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  match_code VARCHAR(50) NOT NULL REFERENCES matches(match_code),
  market VARCHAR(20) NOT NULL,
  probabilities JSONB NOT NULL,
  predicted_outcome VARCHAR(20),
  confidence FLOAT,
  value_bets JSONB,
  model_version VARCHAR(50),
  created_at TIMESTAMP DEFAULT NOW(),
  UNIQUE(match_code, market, model_version)
);
```

**8) `ml_model_runs`**
```sql
CREATE TABLE ml_model_runs (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  model_version VARCHAR(50) UNIQUE NOT NULL,
  trained_at TIMESTAMP DEFAULT NOW(),
  train_size INT,
  test_size INT,
  metrics JSONB,
  feature_count INT,
  notes TEXT
);
```

### E. Günlük Kupon Tablosu
**7) `daily_picks`**
```sql
CREATE TABLE daily_picks (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  pick_date DATE NOT NULL,
  coupon_type VARCHAR(10) NOT NULL,        -- 'safe' | 'value'
  picks JSONB NOT NULL,                     -- 3 pick dizisi
  combined_odds FLOAT NOT NULL,
  avg_wilson FLOAT NOT NULL,
  avg_ev FLOAT NOT NULL,
  results JSONB,                            -- Settle sonrası sonuçlar
  is_settled BOOLEAN DEFAULT FALSE,
  hits_count INT,
  coupon_hit BOOLEAN,
  generated_at TIMESTAMP DEFAULT NOW(),
  settled_at TIMESTAMP,
  UNIQUE(pick_date, coupon_type)
);
```

## 5. Statüler ve İş Kuralları
- `match_queue.status`: `PENDING`, `MONITORING`, `SUCCESS`, `ERROR`, `BAD_DATA`.
- `matches.status`: Mackolik kaynaklı durum (`MS`, `Ert`, vb.).
- `BAD_DATA`: ev/deplasman/lig/sezon eksikse veya geçmiş maçta skor yoksa.
- `MONITORING`: fikstürler ve skoru gelmemiş maçlar.
- `SUCCESS`: skorlar mevcut ve maç tarihi geçmiş.
